# Changelog

All notable changes to this project will be documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [1.2.0] - 2025-12-01

### Added
- Comprehensive airline code enhancement with 413 airline codes
- Airline name display (e.g., "Qantas" instead of "QF")
- Interactive airline code testing interface (airline-code-test.html)
- Backend airline code verification script (test-airline-codes.js)
- Data architecture analysis and scalability framework
- Performance benchmarking tools (benchmark-data-loading.js)
- Database migration templates (migrate-to-sqlite.js)
- Data architecture decision documentation (DATA_ARCHITECTURE_ANALYSIS.md, DATA_ARCHITECTURE_DECISION.md)

### Changed
- Enhanced FlightRadar24 integration to include airline code processing
- Updated frontend JavaScript to display full airline names
- Improved documentation with comprehensive data architecture guidance
- Updated README.md with new features and maintenance instructions

## [1.1.0] - 2025-11-30

### Added
- Aircraft type enhancement with full descriptive names
- Airport code enhancement with city names
- Configuration persistence via JSON files

### Changed
- Admin panel simplified to FlightRadar24 only
- Docker containerization for easy deployment
- Responsive web interface with kiosk mode support

## [1.0.0] - 2025-11-30

### Added
- Initial release of Airview flight tracking application
- Real-time flight tracking using FlightRadar24 API
- Responsive web interface with kiosk mode support
- Admin panel for configuration management
- Docker containerization for easy deployment
- Configuration persistence via JSON files
